以书店业务为背景，构建一套完整的数字化管理解决方案。

通过需求分析明确系统功能边界，涵盖图书管理、客户服务、订单处理、库存监控等核心模块。在数据库设计阶段，创建专用数据库并配置 UTF-8mb4 字符集，确保多语言数据存储兼容性；设计管理人员表（Manager）、图书表（Book）、
客户表（Customer）、付款方式表（Payment）、发货方式表（Shipping）、收支记录表（QuarterlyAccount）、售书记录表（SalesRecord）等核心数据表，
通过外键关联建立实体间逻辑关系。

特别地，实现基于触发器的财务数据自动更新机制，当订单状态变更时实时调整营收记录，保障财务数据的实时性与准确性。
其中售书记录表中详细记录了销售书籍、购书顾客以及售书管理员的等信息，为售书订单的处理提供数据支持，是本次书籍管理系统的最核心功能表。
